# Projeto: Sistema de Gerenciamento de Funcionários

**Descrição:** Criar uma aplicação web moderna para gerenciar funcionários, consumindo uma API REST .NET 8 existente.

## ?? Design e Layout
- Baseie o design visual no site: https://mouts.info/
- Use o mesmo esquema de cores, tipografia e estilo de componentes
- Interface limpa, moderna e responsiva
- Tema profissional com foco em usabilidade

## ?? Tela de Login
- Tela de login simples com campos:
  - Email/Usuário
  - Senha
  - Botão "Entrar"
- **NÃO incluir:**
  - Opção "Lembrar senha"
  - Funcionalidade "Esqueci minha senha"
  - Recuperação de senha
- Validação básica de campos obrigatórios
- Armazenar token JWT no localStorage após login bem-sucedido
- Redirecionar para dashboard após autenticação

## ?? Funcionalidades CRUD de Funcionários

### 1. Listagem de Funcionários
- Tabela responsiva exibindo:
  - ID
  - Nome
  - Email
  - Cargo (Role)
  - Telefone principal
  - Status (Ativo/Inativo)
  - Ações (Visualizar, Editar, Excluir)
- Implementar:
  - Paginação
  - Campo de busca/filtro
  - Botão "Novo Funcionário"
  - Ordenação por colunas

### 2. Visualizar Funcionário
- Modal ou página dedicada mostrando:
  - Informações pessoais completas
  - Cargo/função
  - Lista de telefones
  - Data de criação/atualização
  - Informações do gerente (se houver)
- Botões: "Editar" e "Voltar"

### 3. Criar/Editar Funcionário
- Formulário com campos:
  - Nome completo *
  - Email *
  - CPF
  - Data de nascimento
  - Cargo (select/dropdown) *
  - Telefones (possibilidade de adicionar múltiplos)
  - Gerente (select/autocomplete)
  - Status (Ativo/Inativo)
- Validações:
  - Campos obrigatórios marcados com *
  - Formato de email válido
  - CPF válido (formato brasileiro)
  - Data de nascimento coerente
- Botões: "Salvar" e "Cancelar"

### 4. Excluir Funcionário
- Modal de confirmação antes de excluir
- Mensagem: "Tem certeza que deseja excluir [Nome do Funcionário]?"
- Botões: "Confirmar" e "Cancelar"
- Feedback visual após exclusão

## ?? Integração com API

**Base URL:** `http://localhost:5000/api` (configurável)

**Endpoints da API:**

### Autenticação
```
POST /auth/login
Body: { "email": "string", "password": "string" }
Response: { "token": "jwt_token", "user": {...} }
```

### Funcionários (Employee)
```
GET /employee
Response: Array de funcionários com informações completas

GET /employee/{id}
Response: Dados completos do funcionário específico

POST /employee
Body: {
  "name": "string",
  "email": "string",
  "cpf": "string",
  "birthDate": "2000-01-01",
  "roleId": "guid",
  "managerId": "guid" (opcional),
  "phones": [
    {
      "number": "string",
      "type": "string"
    }
  ]
}

PUT /employee/{id}
Body: Mesmos campos do POST

DELETE /employee/{id}
Response: 204 No Content
```

### Cargos (EmployeeRole)
```
GET /employeerole
Response: Array de cargos disponíveis [{ "id": "guid", "name": "string", "description": "string" }]

GET /employeerole/{id}
Response: Dados do cargo específico
```

## ??? Requisitos Técnicos
- **Framework:** React com TypeScript (ou Next.js se preferir)
- **Estilo:** TailwindCSS ou Shadcn/UI
- **Gerenciamento de Estado:** Context API, Zustand ou Redux Toolkit
- **HTTP Client:** Axios ou Fetch API
- **Roteamento:** React Router DOM
- **Validação:** React Hook Form + Zod
- **Notificações:** React Toastify ou Sonner para feedback de ações
- **Ícones:** Lucide React ou Hero Icons

## ?? Responsividade
- Layout adaptado para:
  - Desktop (1920px+)
  - Tablet (768px - 1024px)
  - Mobile (320px - 767px)
- Menu hambúrguer em dispositivos móveis
- Tabelas com scroll horizontal em telas pequenas

## ?? Fluxo de Navegação
1. **/** ? Tela de login (se não autenticado)
2. **/dashboard** ? Listagem de funcionários (página principal após login)
3. **/employees** ? Mesma listagem (alias)
4. **/employees/new** ? Formulário de criação
5. **/employees/:id** ? Visualização detalhada
6. **/employees/:id/edit** ? Formulário de edição

## ?? Segurança
- Implementar interceptor HTTP para adicionar token JWT em todas as requisições
- Header de autorização: `Authorization: Bearer {token}`
- Redirecionar para login se token expirado (401)
- Proteger rotas privadas com guarda de autenticação
- Limpar localStorage ao fazer logout
- Botão de logout visível no header/sidebar

## ? Diferenciais
- Animações suaves nas transições (Framer Motion)
- Loading states em todas as requisições (spinners, skeleton loaders)
- Tratamento de erros com mensagens amigáveis
- Skeleton loaders durante carregamento inicial
- Confirmação antes de sair de formulários com alterações não salvas
- Debounce no campo de busca
- Estados vazios amigáveis ("Nenhum funcionário cadastrado")
- Feedback visual imediato em todas as ações (toast notifications)

## ?? Estrutura de Dados

### Employee (Funcionário)
```typescript
interface Employee {
  id: string;
  name: string;
  email: string;
  cpf?: string;
  birthDate?: string;
  roleId: string;
  role: EmployeeRole;
  managerId?: string;
  manager?: Manager;
  phones: EmployeePhone[];
  isActive: boolean;
  createdAt: string;
  updatedAt: string;
}
```

### EmployeeRole (Cargo)
```typescript
interface EmployeeRole {
  id: string;
  name: string;
  description?: string;
}
```

### EmployeePhone (Telefone)
```typescript
interface EmployeePhone {
  id?: string;
  number: string;
  type: string; // "Celular", "Comercial", "Residencial"
}
```

### Manager (Gerente)
```typescript
interface Manager {
  id: string;
  name: string;
  email: string;
}
```
